# Purpose

This document captures the primary user stories for SmartChain Nexus using the Agile user story format.

---

## Epic 1 — User Management

### US-001

**As a** System Administrator,

**I want** to create user accounts,

**So that** authorized users can access the system.

### Acceptance Criteria

- User can be created successfully.
- User receives an assigned role.
- Duplicate usernames are not allowed.

---

### US-002

**As a** Registered User,

**I want** to log in securely,

**So that** I can access my dashboard.

### Acceptance Criteria

- Valid credentials allow login.
- Invalid credentials display an error.
- User session is created securely.

---

## Epic 2 — Supplier Management

### US-003

**As a** Procurement Officer,

**I want** to register new suppliers,

**So that** supplier information is centrally managed.

### Acceptance Criteria

- Supplier details are validated.
- Supplier record is saved successfully.
- Duplicate suppliers are prevented.

---

### US-004

**As a** Procurement Manager,

**I want** to view supplier performance,

**So that** I can make informed sourcing decisions.

### Acceptance Criteria

- Supplier KPIs are displayed.
- Performance trends are available.
- Reports can be exported.

---

## Epic 3 — Inventory Management

### US-005

**As a** Warehouse Manager,

**I want** to monitor inventory levels,

**So that** I can avoid stock shortages.

### Acceptance Criteria

- Inventory updates in real time.
- Low-stock alerts are generated.
- Stock movements are recorded.

---

### US-006

**As a** Warehouse Supervisor,

**I want** to transfer stock between warehouses,

**So that** inventory remains balanced across locations.

### Acceptance Criteria

- Source inventory decreases.
- Destination inventory increases.
- Transfer history is recorded.

---

## Epic 4 — Procurement

### US-007

**As a** Buyer,

**I want** to create purchase orders,

**So that** suppliers can fulfill inventory requests.

### Acceptance Criteria

- Purchase order is validated.
- Order receives a unique reference number.
- Order status is tracked.

---

## Epic 5 — Logistics

### US-008

**As a** Logistics Manager,

**I want** to track deliveries,

**So that** shipment performance can be monitored.

### Acceptance Criteria

- Shipment status is visible.
- Delivery delays are highlighted.
- Historical deliveries are retained.

---

## Epic 6 — Executive Reporting

### US-009

**As an** Executive,

**I want** to view KPI dashboards,

**So that** I can make strategic decisions.

### Acceptance Criteria

- Dashboards load quickly.
- KPIs are updated automatically.
- Reports can be exported.
